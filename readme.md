команда запуска с дефолтным .env - docker compose up -d --build
команда запуска с иным ".env" docker compose --env-file ".env"  up -d --build
приложение доступно извне на порту 8080

имеет 2 профиля deploy и dev

dev запускает приложение на локальном tomcat(в моем случае в idee) и коннектится к локальной бд
в случае же с профилем deploy поднимается 2 контейнера, и взаимодействие происходит с ним
для деплоя взаимодействие с БД развернутой в докере выбрал сознательно, так как приложение
носит не коммерческий характер
